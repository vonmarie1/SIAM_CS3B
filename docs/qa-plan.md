# QA Plan

## Test Levels
- Unit Testing: tests individual functions/components
- Integration Testing: tests interaction between modules
- System Testing: tests full application flow

---

## Entry Criteria
- Code is completed and committed
- Feature is merged into dev branch
- No blocking bugs

---

## Exit Criteria
- All unit tests pass
- No critical (S1/S2) bugs remaining
- Feature works as expected

---

## Severity Levels
- S1: Critical (system crash / unusable)
- S2: High (major feature broken)
- S3: Medium (partial issue)
- S4: Low (UI/typo/minor issue)
