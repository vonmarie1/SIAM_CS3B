# Support Plan

## Issue Reporting Process
Users report issues via:
- GitHub Issues
- Email (if applicable)
- Team communication channel

---

## Response Time
- S1 (Critical): within 2–4 hours  
- S2 (High): within 24 hours  
- S3 (Medium): within 2–3 days  
- S4 (Low): next update cycle  

---

## Common Issues
- Chat not connecting → restart server
- Messages not sending → check WebSocket connection
- UI not loading → clear cache or reload page

---

## Escalation Process
1. QA Tester reviews issue  
2. Backend/Frontend developer assigned  
3. Project manager monitors resolution  
4. Critical issues escalated to team lead immediately
